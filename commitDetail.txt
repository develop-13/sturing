커밋 사항: 배포이후의 환경에서 파일 저장 실패하는 이유.
Vercel에서 로컬 파일 시스템 접근은 불가능:

Vercel에서는 서버가 read-only file system을 제공하므로, 
파일 업로드를 위해서는 클라우드 스토리지 서비스를 사용해야 합니다.
로컬 파일 시스템에 직접 저장하는 방식은 사용할 수 없습니다.

배포 환경에서는 로컬 파일 시스템에 쓰는 작업을 할 수 없으므로, 파일 업로드 기능을 외부 클라우드 스토리지 서비스를 이용하여 구현해야 합니다. 
예를 들어 AWS S3를 사용하여 업로드된 파일을 저장하고, 
그 URL을 데이터베이스에 저장하는 방식으로 문제를 해결할 수 있습니다.
라고한다.
